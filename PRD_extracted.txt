Project Title 
Global Money Companion 
Project Overview 
Build a modern, responsive, production-quality web application called Global Money Companion. 
The application helps users understand the value of money across different countries — not only through live 
currency conversion but also through purchasing power comparisons (PPP), historical exchange rates, and a world 
explorer for currencies. 
The application should have a premium fintech-inspired UI and be suitable as a portfolio-quality project. 
Objective 
The application should answer both: 
• ‘100 USD equals how many INR?’ 
• ‘If someone earns ₹25,000/month in India, what salary provides a similar purchasing power in the USA?’ 
Clearly distinguish between exchange-rate conversion and purchasing power (PPP). 
Target Users 
• Students 
• Travelers 
• Professionals moving abroad 
• Remote workers 
• Freelancers 
• Job seekers 
• Anyone interested in global currencies

APIs & Data Sources 
Exchange Rates: Frankfurter API (frankfurter.dev) — free, no key required. 
Country Information: REST Countries API (restcountries.com) — free, no key required. 
PPP Data: World Bank ICP 2021 round — “PPP conversion factor, GDP (LCU per international $)”, indicator 
code PA.NUS.PPP, downloaded as CSV from the World Bank Open Data API.   [TIGHTENED] 
• Store the dataset locally as src/data/ppp-2021.json (converted from the World Bank CSV at build time, not 
fetched live). 
• Record the indicator code and vintage year in the file alongside the data so future updates are traceable. 
Data Contracts 
Shared TypeScript interfaces that every feature must use, so the Converter, Salary Comparison, Favorites, and 
World Explorer stay consistent with one another instead of each inventing its own shape. 
Interface Shape 
Currency code (ISO 4217), name, symbol, decimalDigits 
Country cca2, name, flagUrl, region, subregion, currencies: Currency[] 
ExchangeRate base, target, rate, date, source: 'live' | 'cached' 
PPPEntry countryCode, year, pppConversionFactor, currencyCode, sourceDataset 
SalaryComparison amount, fromCountry, toCountry, exchangeRateResult, pppResult, createdAt 
FavoritePair id, type: 'currency' | 'salary', base, target, label, createdAt 
 
Favorites store the minimal FavoritePair record, not full snapshots — always re-resolve rate/PPP data on load 
so saved favorites never show stale numbers.

Feature 3 — Salary & Purchasing Power Comparison 
Input a salary and compare it with another country. 
Display: 
• Exchange-rate conversion 
• PPP equivalent salary 
• Plain-language explanation of why the values differ 
Always use official PPP conversion factors. 
Acceptance criteria:   [TIGHTENED] 
• If PPP data is missing for the selected country, disable that result panel with an explicit message (see Error 
Handling). 
• Explanation text is generated from a template, not hard-coded per country pair.

Scenario Required Behavior 
Frankfurter API down / times out Fall back to last cached rate in Local Storage; show a 'stale data' badge with 
the cached timestamp. Never show a blank screen. 
REST Countries API down / 
times out 
Fall back to a bundled static JSON snapshot of country/currency metadata 
shipped with the app. 
No internet connection Detect via navigator.onLine + fetch failure; show a persistent offline banner; 
disable actions that require live data. 
Invalid currency code entered Inline validation error under the input; do not fire the API call. 
No search results (World 
Explorer) 
Show an empty-state illustration/message, not a blank list. 
PPP data missing for a country Disable PPP comparison for that country pair and show 'PPP data unavailable 
for [country]' instead of a silent 0 or NaN. 
Loading state Skeleton loaders on cards/tables, not spinners-only, for perceived 
performance.
